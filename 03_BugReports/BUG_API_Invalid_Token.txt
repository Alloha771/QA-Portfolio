BUG ID: BUG_API_007
Заголовок: /api/v1/user возвращает 200 при невалидном токене
Серьёзность: Critical
Приоритет: High
Окружение: Prod, Postman 10.x

Шаги:
1) Выполнить GET /api/v1/user с заголовком Authorization: Bearer abc123 (просрочен)
2) Отправить запрос

Ожидаемый результат:
- HTTP 401 Unauthorized, тело с кодом ошибки (например, {"error":"invalid_token"})

Фактический результат:
- HTTP 200 OK, возвращается объект пользователя

Примечания:
- Потенциальная уязвимость авторизации; проверить middleware/фильтр токена.
